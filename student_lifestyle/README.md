# Student Lifestyle Dataset

## About this dataset

This dataset contains 5,000 fictional survey responses from undergraduate students at a hypothetical university. The responses cover a single academic year that runs from September 1, 2025 to May 15, 2026. Each row corresponds to one student, and each column records a single piece of information about that student or about the student's response to a particular survey item.

The dataset is fully synthetic. It was generated by a Python script that draws values from probability distributions calibrated so that the resulting numbers resemble what would plausibly appear in a real survey of this kind. No real students or institutions are involved in the data, and there is therefore no ethical concern related to consent or to the identification of individuals.

The dataset was prepared for use in DSCI 2410 (Fundamentals of Data Science II), an undergraduate course at the American University in Cairo. The intention is to provide students with a single shared file that is rich enough to support a variety of analytical questions, while being clean enough to be used without specialized domain knowledge. The dataset combines continuous variables, ordinal scales, categorical variables, dates, and binary indicators, which together allow the application of a wide range of common statistical methods.

## File and size

The file is a single comma-separated values (CSV) document with 5,000 rows and 25 columns, encoded in UTF-8. The file size is approximately 600 kilobytes. The unique identifier for each record is the column `student_id`, which takes string values from `S00001` to `S05000`. Although the identifiers run sequentially, the rows are not ordered by `student_id` in any meaningful way, and any analysis should treat the row order as arbitrary.

## Variables

| `admission_year` | integer | Calendar year in which the student first enrolled |
| `home_region` | string | Home neighborhood of the student within Greater Cairo |
| `commuter` | string | `Yes` if the student commutes to campus, `No` otherwise |
| `has_scholarship` | string | `Yes` if the student holds a scholarship, `No` otherwise |
| `major` | string | Academic major of the student (six possible values) |
| `favorite_genre` | string | Self-reported preferred music genre |
| `sleep_hours` | float | Average number of hours slept per night |
| `study_hours` | float | Number of hours spent studying per week |

## Notes on data quality

The dataset has been deliberately seeded with a small number of imperfections, so that it can be used in exercises on data cleaning and quality assessment. Users of the dataset are expected to identify these issues during exploration and to decide how to handle them. The imperfections are listed below.

1. **Missing values in `sleep_hours`.** Approximately three percent of the rows have a missing value in this column. The missingness was introduced at random and is intended to represent non-response on a single survey item.

2. **Structured missingness in the prior-term GPA columns.** The columns `gpa_term1`, `gpa_term2`, and `gpa_term3` are missing for any student who has not yet completed enough academic terms to have a value in those columns. A first-year student therefore has an entry only in `gpa_term4`. A second-year student has entries in `gpa_term3` and `gpa_term4`. The same logic applies to the more senior years. The missingness in these columns is therefore structural rather than random, and any imputation strategy should take that into account.

3. **Outliers in `study_hours`.** A small number of rows report implausibly high values, in the range of approximately one hundred to one hundred forty hours per week. These values represent data-entry errors and should be detected and addressed during cleaning.

4. **Inconsistent text formatting in `major`.** A small fraction of the rows record the major in lowercase rather than in title case. For example, both `Biology` and `biology` may appear. Any analysis that groups records by `major` should normalize the column before grouping.

5. **Invalid values in `coffee_cups`.** A few rows contain a negative value, which is not physically possible. These records should be either removed or corrected before the variable is used in any computation.

## Loading the dataset

The dataset can be loaded directly from this repository using `pandas`:

```python
import pandas as pd

URL = ("https://raw.githubusercontent.com/ahmedmoustafa/datasets/"
       "main/student_lifestyle/student_lifestyle.csv")
data = pd.read_csv(URL)
print(data.shape)
print(data.head())
```

In a Google Colab notebook, no additional setup is required, since the `pandas` package is preinstalled and the URL is publicly accessible.

## Suggested uses

The dataset is suitable for the following types of exercise:

- Descriptive statistics on continuous and categorical variables
- Hypothesis testing using two-sample tests, analysis of variance, and tests of independence
- Correlation and regression analysis
- Visualization of distributions, group comparisons, and temporal trends
- Data cleaning exercises that involve missing values, outliers, and inconsistent formatting
- Reshaping operations on the prior-term GPA columns, such as conversion from wide to long format
